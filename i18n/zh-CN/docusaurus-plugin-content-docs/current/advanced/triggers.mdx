# 触发器自动化

通过监听文件系统变更来自动执行工作流。

## 触发器原理

触发器监听指定目录，在文件事件发生时 **启动工作流**。触发器在后台运行 —— 即便工作流模块没有打开也照常工作。

## 触发器类型

### 文件系统监听器
监听某个目录的文件事件：

| 事件 | 说明 | 用途 |
|-------|-------------|----------|
| **新文件** | 监听目录新增文件 | 自动处理新下载内容 |
| **修改** | 已有文件被修改 | 重新处理更新过的素材 |
| **删除** | 文件被移除 | 清理派生文件 |

### 计划触发器
按指定时间或间隔运行工作流：

- 每天指定时间
- 每小时
- 自定义 cron 表达式

## 配置触发器

1. 打开 **工作流** 模块
2. 选择一个已保存的工作流
3. 点击 **触发器** 标签页
4. 点击 **+ 添加触发器**
5. 配置：
   - **触发器类型**：文件监听 或 计划任务
   - **监听目录**：要监控的目录
   - **文件模式**：要监听的文件（如 `*.png`、`*.jpg`）
   - **工作流**：要执行的工作流
6. 把触发器开关设为 **开启**

## 节流控制

如果监听目录会快速接收大量文件（例如导入相机），请启用节流：

| 设置 | 说明 | 默认 |
|---------|-------------|---------|
| **批次大小** | 每次执行的文件数 | 10 |
| **去抖时间（毫秒）** | 最后一次事件后等待多久才处理 | 2000 |
| **最大并发数** | 同时运行的工作流上限 | 1 |

## 触发器管理

- **启用 / 禁用** —— 不删除触发器，仅切换开关
- **编辑** —— 修改监听目录或关联的工作流
- **查看历史** —— 看触发器处理过哪些文件
- **通知** —— 触发器事件时获得系统托盘提醒

## 示例：自动压缩下载内容

1. 创建一个工作流：`[文件输入] → [压缩] → [保存到目录]`
2. 添加 **新文件** 触发器，监听 `~/Downloads`
3. 文件模式设为 `*.png,*.jpg`
4. 任何保存到 Downloads 的图像都会自动被压缩

## 示例：素材管线

```
相机 SD 卡 → [新文件触发器] → 缩放工作流 → 输出文件夹 → [新文件触发器] → 压缩工作流 → 最终素材
```

把多个触发器串成复杂管线 —— 每个触发器各自启动自己的工作流。
