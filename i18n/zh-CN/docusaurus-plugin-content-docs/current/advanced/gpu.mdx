# GPU 加速

Pixore 支持对兼容操作进行 GPU 加速。

## 支持的后端

| 后端 | 库 | 状态 |
|---------|---------|--------|
| **OpenCL** | PyOpenCL | ✅ 支持 |
| **CUDA** | CuPy | ✅ 可选 |

## 启用 GPU 加速

1. 打开 **设置 → 性能**
2. 勾选 **启用 GPU 加速**
3. 选择偏好的后端（OpenCL 或 CUDA）
4. 重启 Pixore

Pixore 启动时会自动检测可用 GPU。如果没找到兼容 GPU，该选项会变灰。

## 哪些操作会被加速

GPU 加速适用于受益于并行计算的操作：

- **批量图像缩放** —— 用 OpenCL 最高可快 10 倍
- **位深转换** —— 在 GPU 上做 Floyd-Steinberg 抖动
- **图像滤镜** —— 卷积类滤镜（模糊、锐化、边缘检测）
- **色彩空间转换** —— HSL、曲线、色阶调整
- **AI 模型推理** —— 当本地模型支持 GPU 时

## 性能对比

测试条件：1000 张图（1920×1080）缩放到 512×512：

| 操作 | CPU（8 核） | GPU（OpenCL） | 加速比 |
|-----------|---------------|--------------|---------|
| Lanczos 缩放 | 45.2 秒 | 4.8 秒 | **9.4×** |
| Floyd-Steinberg | 32.1 秒 | 5.2 秒 | **6.2×** |
| 高斯模糊 | 18.7 秒 | 2.1 秒 | **8.9×** |

## 故障排查

### 检测不到 GPU
- 更新显卡驱动
- 安装 OpenCL 运行时（Intel/NVIDIA/AMD）
- 用 `python -c "import pyopencl; print(pyopencl.get_platforms())"` 验证

### 显存不足错误
- 在「设置 → 性能 → 节流」减小批次
- 关闭其他占用 GPU 的应用
- 试试另一个后端（OpenCL 与 CUDA 互换）

### CPU 回退
GPU 加速过程中失败时，Pixore 会自动回退到 CPU 处理。详情请查阅日志。
