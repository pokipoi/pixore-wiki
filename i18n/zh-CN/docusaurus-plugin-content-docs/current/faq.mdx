# 常见问题

## 概述

### Pixore 是什么？
Pixore 是一款桌面图像处理工作站，由八个专用模块和基于节点的可视化工作流引擎组成，可替代多个单一用途的小工具。

### Pixore 收费吗？
完全免费，并以 GPL v3 协议开源。

### 支持哪些平台？
完整支持 Windows 10+。macOS 与 Linux 原生包正在规划中。

---

## 安装

### Windows 上提示「缺少 DLL」？
请安装 [Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)。

### 「No module named X」错误？
重新执行 `pip install -r requirements.txt`。首次启动时 `envinit.py` 会自动安装缺失依赖。

### 没装 Python 也能用 Pixore 吗？
可以，从 [Releases](https://github.com/pokipoi/pixore/releases) 下载预编译安装包即可。

## 模块

### 「批量缩放」模块和工作流里的「PIL Resize」节点有何区别？
**批量缩放** 使用 Pixore 优化过的管线，带智能预裁与并行处理；**PIL Resize** 是 Pillow 的轻量封装，更简单但功能不足。

### 可以无损压缩吗？
- **PNG**：把 pngquant 质量设到 100，进入无损回退模式
- **JPEG**：MozJPEG 用质量 100（理论上仍是有损，但视觉上无损）
- **WebP**：在压缩设置里启用无损模式

### 切割模块支持透明背景吗？
支持。带透明背景的切割区域会导出为 RGBA PNG。开启 **Always-Trim Output** 可自动裁剪透明边缘。

### AI 重命名能离线工作吗？
可以！配合 **Ollama** 或 **LM Studio** 使用本地模型，无需联网。

## 工作流与节点

### 一个工作流最多能放多少个节点？
没有硬性限制，性能取决于硬件与节点复杂度。50+ 节点的工作流在普通硬件上也能流畅运行。

### 自定义节点能用 Python 第三方库吗？
可以。把依赖写进 `requirements.txt`，节点加载时会自动安装。

### 社区节点装在哪？
存放在 Pixore 数据目录下的 `nodes_library/community/`。

## 性能

### Pixore 跑得很慢，怎么办？
1. 在「设置」中开启 GPU 加速
2. 在「设置 → 性能」中提高进程数
3. 大批量任务降低分块大小
4. 关闭其他占内存的应用

### Pixore 会用我的 GPU 吗？
会的，缩放、抖动、滤镜等兼容操作都会用 GPU。在「设置 → 性能 → GPU 加速」开启。

### 需要多少内存？
- 基础使用：4 GB
- AI 节点 + 大批量：最低 8 GB，推荐 16 GB

## 故障排查

### 日志在哪？
Pixore 安装目录下的 `logs/`，最新日志为 `pixore.log`。

### 工作流总是失败，如何排查？
1. 在设置中开启详细日志
2. 用 **代码视图** 查看生成的代码
3. 查看每个节点执行期间的日志输出
4. 单独运行节点以定位问题

### 如何重置所有设置？
删除 Pixore 数据目录下的 `config.json`，下次启动会用默认值重新生成。
