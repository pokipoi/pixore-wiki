架构设计

Pixore 采用 分层架构，主程序与节点生态彻底解耦。

分层概览​

┌─────────────────────────────────────────────────────────────┐
│                         界面层                              │
│              PyQt5 + PyQt-Fluent-Widgets                    │
│   首页 · 缩放 · 拖拽 · 压缩 · 位深 · 切割                   │
│        重命名 · 工作流 · 设置 · 对话框                      │
├─────────────────────────────────────────────────────────────┤
│                       应用服务层                            │
│   触发管理器 · 信号总线 · 配置 · 主题管理                   │
│   进程池 · 异步图像加载器 · 缓存管理                        │
├─────────────────────────────────────────────────────────────┤
│                       工作流引擎                            │
│   拓扑调度器 · 执行上下文 · 节流控制                        │
├─────────────────────────────────────────────────────────────┤
│                       节点加载器                            │
│      清单解析 · 依赖解析 · 市场对接                         │
├─────────────────────────────────────────────────────────────┤
│                       节点库                                │
│   内置节点 · 社区节点 · 赞助专业节点                        │
├─────────────────────────────────────────────────────────────┤
│                       外部工具                              │
│       MozJPEG · Guetzli · pngquant · ImageMagick            │
└─────────────────────────────────────────────────────────────┘

界面层​

最顶层负责所有用户交互：

  • PyQt5 提供原生窗口与控件系统
  • PyQt-Fluent-Widgets 带来现代 Fluent Design 视觉
  • QFramelessWindow 实现自定义无边框窗口，支持置顶
  • 每个模块都是独立的界面组件，加载到导航栈中

应用服务层​

为整个应用提供横切能力：

服务作用
触发管理器监听文件系统事件，调度工作流执行
信号总线基于 Qt 信号的模块间解耦通信
配置管理持久化设置（主题、路径、GPU、默认值）
主题管理浅/深主题切换，支持系统偏好检测
进程池多进程任务执行，支持进度追踪与取消
异步图像加载器表格与网格视图的非阻塞缩略图生成
缓存管理缩略图、节点输出与模型结果的 LRU 缓存

工作流引擎​

核心执行系统：

拓扑调度器​

节点构成一张 有向无环图（DAG）。调度器：

  1. 计算拓扑顺序
  2. 在每一层识别可并行的节点
  3. 通过进程池分发任务
  4. 跟踪依赖，把输出送给下游节点

执行上下文​

工作流运行期间所有节点共享的 dict 键值存储。节点可以：

  • 写入 中间结果：ctx["preview_path"] = "/tmp/thumb.png"
  • 读取 上游结果：data = ctx.get("image_data")
  • 上下文值在单次工作流运行的生命周期内有效

节流控制​

针对大批量文件处理：

  • 可配置的 分块大小（例如每次处理 10 个文件）
  • 可配置的 并发度（同时处理多少块）
  • 自动分批输入，避免内存溢出

节点加载器​

负责发现、校验与解析节点：

  1. 发现 —— 扫描 nodes_library/ 与已安装的市场目录
  2. 清单解析 —— 校验 manifest.json 的必需字段
  3. 依赖解析 —— 检查 Python 导入与外部工具可用性
  4. 注册 —— 把合法节点加入类型注册表，供工作流使用

节点库​

节点分三档：

层级位置说明
内置nodes_library/Pixore 自带的核心节点
社区市场安装来自社区索引的用户贡献节点
专业版赞助者专享高级 AI 节点与专业模板

外部工具​

Pixore 内置或集成：

  • MozJPEG —— Google 出品的极致体积优先 JPEG 编码器
  • Guetzli —— 知觉质量优化（速度慢，但画质极佳）
  • pngquant —— 基于质量目标做调色板缩减的 PNG 压缩工具
  • ImageMagick —— 万能瑞士军刀，处理格式转换与批量操作

技术栈​

层级所用技术
GUIPyQt5 · PyQt-Fluent-Widgets · QFramelessWindow
图像处理Pillow · NumPy · OpenCV · SciPy · Shapely
AIOpenAI SDK · rembg · 自研多模态集成
压缩MozJPEG · Guetzli · pngquant · ImageMagick
工作流引擎自研拓扑调度器 + 执行上下文
GPU 加速PyOpenCL · CuPy（可选）
系统集成pywin32（Windows）· Send2Trash · QFileSystemWatcher
